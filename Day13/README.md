# Day 13: 陣列上的極速切割 - 二元搜尋法 (Binary Search)

## 1. 什麼是二元搜尋法？
就像查字典一樣，每次從中間翻開。如果目標比中間大，就直接把前半本撕掉；反之亦然。
* **前提條件**：陣列必須是**已經排序好**的狀態。
* **時間複雜度**：每次對半砍，速度是極速的 **$O(\log N)$**。

## 2. 🚨 經典面試陷阱：溢位 (Integer Overflow)
一般人常寫的 `mid = (low + high) / 2` 藏著一個潛伏在 Google 演算法中長達 9 年的 Bug！當陣列極大時，`low + high` 會超過 C 語言 `int` 的極限 (21.4 億) 變成負數，引發記憶體崩潰。
👉 **安全寫法**：
```c
int mid = low + (high - low) / 2;
```
先算距離再減半，保證絕對不會溢位。

## 3. 🚀 微處理器極限優化：右移取代除法
在 CPU 底層，除法 (`/`) 是一個極度耗時的運算。我們可以利用二進位的特性，用「右移 1 位」來完美取代「除以 2」。
👉 **極速寫法 (Linux Kernel 愛用)**：
```c
int mid = low + ((high - low) >> 1);
```

## 4. 為什麼是無條件捨去？
在 C 語言的整數運算中，`1 / 2` 會無條件捨去變成 `0`。這個特性確保了當 `low` 和 `high` 相鄰（只剩 2 個元素）時，`mid` 會精準落在 `low` 的位置，使得範圍能夠順利收斂，避免程式陷入無限迴圈。

## 5. 尋找左邊界 (Lower Bound)
如果陣列裡有大量重複的數字（如 `10, 20, 20, 20, 30`），且面試官要求回傳「第一個出現的 20」。
* **錯誤解法**：找到後用 `while` 往左邊一格一格找。這會讓效能退化回 $O(N)$。
* **正確解法**：找到目標時，把位址記錄下來，然後**強制執行 `high = mid - 1`**，繼續對左半部進行二分搜尋，直到迴圈結束。這樣就能死守 $O(\log N)$ 的極致效能！
