# Day 4: 函式指標、回呼架構與底層中斷大解密

## 1. 函式指標 (Function Pointer) 與 指標函式 (Pointer Function)
在 C 語言中，這兩個名詞極度容易搞混，必須從宣告的「括號」來分辨：
* **函式指標**：`int (*fp)(int, int);`
  * **本質**：是一個「指標變數」，專門用來裝某個函式的記憶體位址。
* **指標函式**：`int* func(int, int);`
  * **本質**：是一個「正港的函式」，執行完畢後會回傳一個指標 (`int*`) 給你（例如 `malloc`）。

## 2. 回呼函式 (Callback Function) 與驅動架構
函式指標最經典的應用是在作業系統底層與硬體驅動中：
* **解耦 (Decoupling)**：底層硬體（如按鈕 ISR）不需要知道按下後要做什麼，它只留下一個「空位（函式指標）」。
* **註冊 (Register)**：高階應用層把想做的行為（如 `turn_on_led`）的位址填入這個空位。
* **防禦性設計**：在執行 Callback 之前，**絕對要檢查 `if (callback != NULL)`**。如果沒檢查就執行空指標，CPU 會跳躍到 `0x00000000` 執行未知的非法指令，在 STM32 中會立刻觸發 **HardFault (硬體錯誤)**，導致系統完全卡死崩潰。

## 3. 多型與 `return a - b;` 的排序魔法
函式指標賦予了 C 語言「多型」的能力，例如內建的 `qsort`。
* 排序引擎不預設規則，而是要求我們傳入一個比較函式（函式指標）。
* **數學神技**：排序引擎只看回傳值的正負號。
  * 若要**由小排到大 (Ascending)**：只要寫 `return a - b;`。若 a 較小，回傳負數，引擎就會把 a 乖乖留在左邊。
  * 若要**由大排到小 (Descending)**：只要寫 `return b - a;`。

## 4. 魔王考題：解讀 `signal` 的神級宣告
```c
void (*signal(int sig, void (*func)(int)))(int);
```
這行宣告展現了指標的終極奧義，利用「由內而外剝洋蔥」的方式解讀：
1. **核心**：`signal` 是一個**函式**。
2. **參數**：它接收兩個參數。第一個是普通的整數 `sig`，第二個是函式指標 `void (*func)(int)`。
3. **回傳值**：把中間的核心與參數拔掉後，剩下 `void (*)(int)`。這代表 `signal` 執行完後，會**回傳另一個函式指標**。

* **實戰應用**：我們可以用它來攔截作業系統的 `Ctrl+C` (`SIGINT`) 訊號，把原本「關閉程式」的行為，偷換成我們自己定義的 Callback 函式，而它回傳的舊函式指標，可以讓我們未來再把預設行為改回來。
