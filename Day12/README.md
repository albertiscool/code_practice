# Day 12: 遞迴的終極舞台 - 二元搜尋樹 (Binary Search Tree)

## 1. 什麼是二元搜尋樹 (BST)？
BST 是一種非線性資料結構，每個節點擁有最多兩個分支 (`left` 與 `right`)。
它的核心規則非常直覺：**「比我小往左走，比我大往右走」**。這種特性讓它成為「遞迴 (Recursion)」最完美的發揮舞台。

## 2. 遞迴在樹中的三大發揮
* **插入新節點 (`insert_node`)**：不斷比對大小，把工作往下丟，直到遇見 `NULL` 才安插新節點。
* **排序印出 (In-order Traversal)**：採用「左小弟 $\rightarrow$ 老大 $\rightarrow$ 右小弟」的順序印出，結果保證是**從小到大完美排序**。
* **記憶體釋放 (Post-order Traversal)**：拆大樓必須從頂樓拆！釋放節點時，必須按照「左 $\rightarrow$ 右 $\rightarrow$ 根」的順序。若先 `free(root)` 會導致左右子樹的位址遺失，引發 Memory Leak 與 Segmentation Fault。

## 3. 致命退化與時間複雜度
* **極速神話**：在完美的平衡狀態下，BST 每次尋找都能將範圍「對半砍」，時間複雜度為極速的 **$O(\log N)$**。
* **最倒楣狀況 (Degenerate)**：若依序插入已排序的數字 (例如 `10, 20, 30, 40`)，樹會長成一條直線，物理結構徹底退化為「單向連結串列」。此時時間複雜度慘跌為 **$O(N)$**。
* **業界解法**：為了防止退化，業界發明了「紅黑樹 (Red-Black Tree)」等自平衡樹，在插入時自動旋轉扭正，死守 $O(\log N)$ 的極限速度。

## 4. 實戰白板題：`search` 的兩大陷阱
當我們手寫尋找節點的遞迴函式時，面試官一定會緊盯兩個地方：
1. **忘記底層防護**：沒有寫 `if (root == NULL)` 防呆，導致走到死路時引發當機。
2. **忘記交差 (Missing Return)**：寫了 `search(root->left, key);` 卻忘記在前面加上 `return`。這會導致小弟找到的寶物被丟掉，無法傳遞回最上層的 `main` 函式。

> **💡 C 語言指標設計鐵則**：如果尋找函式 (`search`) 找不到目標，**永遠一律回傳 `NULL`**。這讓外部呼叫者能輕鬆利用 `if (result != NULL)` 進行優雅的安全判斷。
