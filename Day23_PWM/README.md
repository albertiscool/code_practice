# Day 23: 計時器 (Timer) 與 PWM 動態調變架構

今天我們解析了微處理器用來控制類比世界（如馬達變速、LED 呼吸燈調光）的最強武器：**Timer 與 PWM (脈衝寬度調變)**，並深入硬體底層，揭開了保護電路板不被燒毀的關鍵機制！

## 1. 📐 PWM 的黃金三角暫存器
產生精準的 PWM 波形，全靠三個硬體暫存器的連動比對：
* **`CNT` (Counter, 計數器)**：Timer 的心臟，每一次硬體時鐘敲擊就 `+1`。
* **`ARR` (Auto-Reload Register, 天花板)**：決定波形的 **頻率 (週期)**。當 `CNT` 數到 `ARR` 時會自動歸零。
* **`CCR` (Capture Compare Register, 門檻)**：決定波形的 **工作週期 (Duty Cycle)**。
  * 當 `CNT < CCR` 時，硬體比較器強制輸出 `1` (High)。
  * 當 `CNT >= CCR` 時，強制輸出 `0` (Low)。
* **動態調速奧義**：若要讓馬達加速，完全不用停下 Timer，只要在程式中修改 `CCR` 的數值，PWM 佔空比就會瞬間改變。

## 2. 🛡️ 護身符一：死區時間 (Dead-Time Insertion)
在工業控制 (如 H 橋馬達驅動) 中，我們常需要兩路「互補」的 PWM 訊號。
* **致命災難 (Shoot-Through, 上下橋直通)**：因為電晶體「關閉」的速度比「打開」慢，如果切換時不加延遲，會發生上橋還沒完全關閉，下橋就打開的狀況，導致 VCC 直接短路到 GND，電路板當場炸毀冒白煙。
* **硬體解法**：開啟硬體的 **死區時間 (Dead-Time)**。硬體會在切換時強制插入幾百奈秒的「雙方皆關閉」空白期，確認前一顆電晶體死透了，才打開下一顆。

## 3. 👻 護身符二：影子暫存器 (Shadow Register) 與 Preload 機制
如果在程式執行一半，`CNT` 數到一半時，我們突然改寫 `ARR`，會讓 `CNT` 錯過天花板，引發計數器失控與波形暴走 (Glitch)。
* **Preload 緩衝區**：我們程式寫入的 `TIM->ARR` 或 `TIM->CCR`，其實只是寫入表面的「預裝載暫存器」。寫入當下**不會立刻生效**。
* **Update Event (更新事件)**：必須等到現在這一輪的週期跑完，`CNT` 歸零的那個瞬間，硬體才會一口氣把表面暫存器的新數值「傾印 (Copy)」到晶片深處真正在運作的 **影子暫存器 (Shadow Register)** 中。
* **優勢**：保證每一次的頻率與速度改變，都完美銜接在下一個週期的起點，絕不會產生破圖波形！
