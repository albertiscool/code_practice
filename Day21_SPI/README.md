# Day 21: SPI 通訊協定 (全雙工移位與拓撲架構)

今天我們進入了地表最快、最硬派的同步通訊協定：**SPI (Serial Peripheral Interface)**。
我們破解了它與 I2C / UART 最大不同的「全雙工物理限制」，以及硬體拓撲設計上的哲學。

## 1. 🏓 SPI 的靈魂：全雙工移位暫存器 (Shift Register)
在 SPI 的世界裡，**沒有單純的「寫 (Write)」，也沒有單純的「讀 (Read)」，永遠只有「交換 (Exchange)」**。
* Master 和 Slave 內部的 Shift Register 透過 `MOSI` 與 `MISO` 兩根線頭尾相連，形成一個封閉的圓環。
* 每當 SCLK 敲擊一次，雙方的 Bit 就會互推。8 個 Clock 之後，雙方手上的 8-bit 資料完美互換。
* **面試必考題**：如何從感測器讀取資料？
  > 答：大老闆 (Master) 必須主動送出 8 顆「空包彈 (Dummy Data, 如 `0xFF` 或 `0x00`)」，利用硬體推擠的力量，把感測器內部的真實數據硬生生地「擠」出來給自己。

## 2. ⏳ 晶片內部的時間差 (先後順序)
當 Master 發送指令 (如 `0x9F`) 時，Slave 內部的真實運作順序：
1. **外線物理交換**：Master 的指令被推進 Slave 的移位暫存器。
2. **大腦光速偷天換日**：收滿 8-bit 後，Slave 的內部邏輯瞬間解碼指令，並找出對應的結果 (如 ID 號碼)，**強制覆蓋**掉移位暫存器裡的垃圾資料。
3. **二次物理交換**：Master 發送空包彈，將剛才大腦放好的真實數據擠出來。

## 3. 🌐 多設備架構對決：獨立片選 vs 雛菊鏈
* **獨立片選 (Independent CS)**：
  每台 Slave 都有獨立的 CS 線。好處是可以極速點名任意裝置 (低延遲)，缺點是極度浪費 MCU 的 GPIO 腳位 ($3 + N$ 根腳位)。
* **雛菊鏈 (Daisy Chain)**：
  全部 Slave 串聯成一根超級長管子。永遠只需要 **4 根腳位**。好處是節省硬體資源，適合整批同步更新 (如 LED 燈牆)；缺點是想更新最後一顆必須把前面全推一遍，且只要中間斷一顆，後面全部癱瘓。
* **現代硬體救星 (如 WS2815 燈條)**：
  加入「雙路訊號備援 (Dual-Signal Wires)」。透過額外的 `BI` (Backup In) 腳位，當前一顆晶片燒毀時，能自動越級讀取前前一顆的訊號，完美解決雛菊鏈的單點故障弱點。

## 4. ⏱️ 搞懂 SPI 的四種模式 (CPOL / CPHA)
SPI 允許硬體配置不同的時脈極性（CPOL）與時脈相位（CPHA），總共組成了 4 種 SPI Mode：

* **模式 0 (CPOL=0，CPHA=0)**：空閒狀態下，SCLK 為低電平，資料取樣在第一個邊緣，即 SCLK 由低電平跳變到高電平，因此資料採樣在上升沿（準備資料），資料發送在下降沿。
* **模式 1 (CPOL=0，CPHA=1)**：空閒狀態下，SCLK 為低電平，在第一個沿發送數據，即 SCLK 由低電平跳變到高電平，因此在下降沿進行數據採樣，在上升沿發送數據。
* **模式 2 (CPOL=1，CPHA=0)**：空閒狀態下，SCLK 為高電平，資料擷取是在第一個邊緣，即 SCLK 由高電平跳變為低電平，所以資料收集是在下降沿，資料傳輸是在上升沿發送。
* **模式 3 (CPOL=1，CPHA=1)**：空閒狀態下，SCLK 為高電平，數據傳輸是在第一個沿發送，即 SCLK 由高電平跳變為低電平，因此在上升沿採集數據，在下降沿傳輸數據。
