# Day 10: 演算法極致 - 遞迴特訓與龜兔賽跑演算法

## 1. 遞迴 (Recursion) 實戰：費氏數列
完美套用「無恥工作分配法」的三大步驟：
1. **終止條件 (Base Case)**：`if (n == 0 || n == 1) return n;` (老大自己做)
2. **信任小弟 (Delegate)**：`fibonacci(n-1)` 與 `fibonacci(n-2)` (切小問題)
3. **收尾工作**：將小弟的回傳值相加 `return prev1 + prev2;`

## 2. 揪出死迴圈：龜兔賽跑演算法 (快慢指標)
此為 LeetCode 141 經典題，用以揪出連結串列中可能導致無限迴圈的惡意指標。
* **烏龜 (`slow`)**：一次走 1 步。
* **兔子 (`fast`)**：一次走 2 步。
* **原理**：若串列存在迴圈，兔子最終一定會從背後「倒追 (套圈)」並撞上烏龜 (`slow == fast`)。

## 3. 邊界防護與當機 (Segmentation Fault) 陷阱
判斷式 `while (fast != NULL && fast->next != NULL)` 兩者缺一不可：
* **`fast != NULL`**：防護「偶數」節點的串列，防止兔子跑到終點後踩空。
* **`fast->next != NULL`**：防護「奇數」節點的串列，防止兔子在跳躍的第二步墜崖。
C 語言利用 `&&` 的「短路求值」特性，完美避免了越界存取。

## 4. 數學思維：為什麼兔子只能走 2 步？
因為兔子的相對追趕速度是 **2 - 1 = 1**。這保證了兩者的距離會穩穩地每次減 1，最終精準降為 0 (完美相撞)。若兔子走 3 步，相對速度為 2，在距離為奇數時兔子會直接「飛躍」烏龜，導致檢測時間大幅拉長甚至失敗。

## 5. 魔王題：如何找出迴圈的「起點」？
利用數學推導出相撞時的距離公式：**$L = n \times C - K$**
(從起點走到入口的距離) = (從相撞點繼續往前走到入口的距離)。
👉 **解法**：在相撞的那一刻，把其中一個指標瞬間傳送回 `Head`，然後兩個指標都降速成「每次走 1 步」。當它們下一次相遇時，相撞的地點必然就是死迴圈的入口節點！
