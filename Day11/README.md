# Day 11: 系統底層的心臟 - 環狀佇列 (Circular Queue)

## 1. 環狀佇列的核心魔法
為了解決傳統陣列在移除資料時需要「全體往前搬移」的效能瓶頸，我們將陣列的頭尾相連。利用取餘數運算子 (`%`)，讓指標在走到陣列底端時瞬間回到開頭，達成無限繞圈的效果：
```c
q->tail = (q->tail + 1) % QUEUE_SIZE;
```

## 2. 結構體參數：為什麼必須傳遞指標 (`*q`)？
* **修改正本 (Modifiability)**：C 語言是 Pass-by-Value，若不傳遞記憶體位址，函式只會修改影印本，無法真正改變佇列狀態。
* **極致效能 (Performance)**：結構體內包含龐大陣列，傳遞指標只需複製 8 Bytes 的地址，避免了將整個陣列塞入 Stack 的巨大時間與空間浪費。

## 3. 面試必考：判斷「空」與「滿」的三大流派
當 `head == tail` 時，系統會陷入「全空」還是「全滿」的迷惘。業界有三種解法：
1. **犧牲一格流 (最速)**：刻意留一個空位不裝資料。若尾巴的下一格是頭 `(tail + 1) % SIZE == head`，即判定為滿。
2. **計數器流 (最直覺)**：新增一個 `count` 變數，入列 `+1`、出列 `-1`。不浪費空間，但增加 CPU 數學運算。
3. **最後操作標記流 (最靈活)**：新增一個 `bool` 紀錄最後一次動作。若撞在一起時，上次動作是「寫入」就是全滿，是「讀取」就是全空。

## 4. 微處理器極限優化：取代 `%` 的位元魔法
在微處理器 (MCU) 中，除法 (`%`) 運算極度消耗 CPU 時脈。
資深工程師會將 `QUEUE_SIZE` 設定為 **2 的冪次方**（如 16, 64, 256）。此時可以使用位元運算 **`& (SIZE - 1)`** 來完美取代 `% SIZE`：
```c
// 超快速的優化寫法 (以 size = 256 為例)
q->tail = (q->tail + 1) & 255;
```
這種寫法利用二進位遮罩過濾掉進位，只需 1 個時脈週期即可完成運算，速度大幅飆升！
