# Day 24: DMA (直接記憶體存取) 與硬體底層架構大解密

今天我們學會了釋放微處理器終極效能的最強神兵：**DMA (Direct Memory Access)**。
我們從基本的 I/O 卸載，一路探討到電競外掛的物理原理，甚至解開了高階晶片最致命的「快取一致性」當機陷阱！

## 1. 🚚 為什麼需要 DMA？ (大廚與外送員)
如果沒有 DMA，每次收到 UART 傳來的一個字元，CPU 都必須中斷手邊的高階演算法去搬運資料，導致系統效能癱瘓。
* **DMA 的功用**：它是一個純硬體的搬運工。CPU 只要交代三件事：「源頭位址 (`CPAR`)」、「目的地位址 (`CMAR`)」、「搬運總量 (`CNDTR`)」。
* 交代完後，硬體會自動在背景透過匯流排搬運資料，CPU 可以全神貫注處理高複雜度的運算。直到全部搬完，DMA 才會發出中斷通知 CPU。

## 2. ⚔️ 硬體外掛原理 (PCIe DMA Cheats)
為什麼 DMA 設備可以用來做槍戰外掛，且防作弊軟體抓不到？
因為插在 PCIe 插槽上的 DMA 介面卡，擁有在物理層級「繞過 CPU 與作業系統，直接讀取實體 RAM」的特權。由於不經過作業系統，駐守在核心層的防作弊軟體（警察）完全瞎了眼，無法察覺記憶體正在被外部電腦竊取。

## 3. 🏓 雙緩衝區串流 (Ping-Pong Buffer)
在接收無間斷的串流資料（如高頻音訊、影像）時，我們會讓 DMA 跑在「循環模式 (Circular Mode)」不斷覆蓋陣列。
為了防止 CPU 讀取到一半被 DMA 覆蓋（鬼抓人），我們會利用兩個中斷：
* **過半中斷 (Half-Transfer, HT)**：DMA 填滿前一半 (Ping 區) 時觸發。CPU 可安全讀取前一半，因 DMA 正忙著寫入後一半。
* **完成中斷 (Transfer Complete, TC)**：DMA 填滿後一半 (Pong 區) 時觸發。CPU 可安全讀取後一半，因 DMA 已繞回開頭寫入前一半。
這實現了「無限長度、不間斷、零遺失」的串流處理架構。

## 4. 💣 高階晶片地雷：快取一致性 (Cache Coherency)
在帶有硬體快取 (Hardware Cache) 的高階晶片（如 Cortex-M7、Jetson Nano）中，`volatile` 關鍵字只能防編譯器優化，**絕對防不住硬體快取**！
* **災難發生**：DMA 在背景悄悄把最新資料寫入實體 RAM，但 CPU 讀取時，硬體快取攔截了請求，並丟出以前備份的舊資料 (髒數據)。
* **防禦解法 1 (手動洗快取)**：在 CPU 讀取前，強制呼叫 `SCB_InvalidateDCache()` (快取無效化)，撕毀快取舊檔，強迫從 RAM 讀取最新資料。
* **防禦解法 2 (設定禁區)**：利用 MPU (記憶體保護單元) 將 DMA 使用的 RAM 區塊劃分為「不允許快取 (Non-Cacheable Region)」，讓讀寫直接穿透在實體 RAM 上。
