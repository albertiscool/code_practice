# Day 22: SAR ADC 類比數位轉換與硬體二分搜尋法

今天我們探索了微處理器讀取真實世界訊號的第一道關卡：**ADC (類比數位轉換器)**。
我們發現了演算法課程學的「二分搜尋法」，竟然就是硬體 SAR ADC 內部的運作靈魂，並且探討了取樣時間與資料對齊的硬體優化技巧！

## 1. 🔍 二分搜尋法的邊界與防溢位 (Day 22 Review)
在撰寫尋找「左邊界（第一個出現的數字）」的二分搜尋法時，有三大必考重點：
* **防溢位 (Overflow Safe)**：`mid` 的計算絕對不能寫 `(low + high) / 2`，因為相加可能會造成整數溢位。正確寫法是：`mid = low + ((high - low) >> 1)`。
* **互斥邏輯**：必須使用 `if` -> `else if` -> `else`，否則會發生連續進入多個條件區塊的恐怖 Bug。
* **左邊界策略 (Save and Keep Going)**：當 `arr[mid] == target` 時，先用變數把 `mid` 存起來，然後強制 `high = mid - 1`，繼續往左邊找有沒有更早出現的目標。

## 2. ⚡ SAR ADC (逐次逼近型 ADC)：硬體版二分搜尋
STM32 晶片內部廣泛使用的 SAR ADC，其實就是在做二分搜尋！
* **運作原理**：從最高位元 (MSB) 開始猜，每次設為 1，並用內建 DAC 轉成電壓跟外部訊號比較。沒超過就保留 1，超過就清回 0。
* **極致效率**：因為每次都砍半猜測，所以一個 12-bit 的 ADC，無論電壓多少，永遠只要精準的 12 個 Clock 就能找出 0~4095 中的答案。
* **量化誤差 (Quantization Error)**：12-bit 的解析度極限是 $3.3V \div 4096 \approx 0.0008V$。只要是 ADC 就一定有量化誤差，無法 100% 精準。

## 3. ⏳ 取樣時間 (Sampling Time) 與 RC 阻抗匹配
ADC 測量前，必須先打開開關讓外部電壓灌滿內部的「取樣小電容 (Sample-and-Hold Capacitor)」。
* **致命陷阱**：如果感測器輸出阻抗很大（給電能力弱），電容充電速度會變超慢（RC 時間常數）。
* **軟體解法**：在暫存器把 ADC 的「取樣時間 (Sampling Time)」調大，多給它一點時間充電，避免讀出偏低的錯誤電壓。
* **硬體解法**：加上運算放大器 (Op-Amp) 作為電壓隨耦器，降低阻抗，瞬間灌滿電容。

## 4. 🗜️ 資料對齊 (Data Alignment) 的極致效能優化
12-bit 的資料必須存在 16-bit 的暫存器中。
* **右對齊 (預設)**：精準讀取 12-bit，但必須操作 2 個 Bytes。
* **左對齊 (效能狂人模式)**：將前 8 個最重要的 bit 擠到 16-bit 暫存器的「高 8 位元 (High Byte)」。
  * **應用場景**：當只想要「取個大概 (8-bit 解析度即可)」時（例如判斷白天黑夜），直接強制讀取高 8 位元位址即可。
  * **好處**：RAM 消耗瞬間減半、UART 傳輸速度翻倍、且硬體直接幫你做好位元平移，CPU 不用浪費時間算數學！
