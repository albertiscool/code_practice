# Day 7: 韌體開發核心 - 位元運算 (Bitwise Operations) 

## 1. 操控暫存器的「四大天王」
在不影響硬體其他腳位的前提下，精準操控單一 Bit 的標準寫法：
* **啟動 (Set Bit 1)**：`REG |= (1 << n);`
  * 利用 `|` (OR) 有 1 則 1 的特性，將第 n 個 bit 強制變 1。
* **反轉 (Toggle Bit)**：`REG ^= (1 << n);`
  * 利用 `^` (XOR) 相同為 0、不同為 1 的特性，遇到 1 就翻轉狀態，常用於閃爍 LED。
* **檢查 (Check Bit)**：`if (REG & (1 << n))`
  * 利用 `&` (AND) 濾掉其他雜訊，如果結果大於 0 代表該 bit 為 1 (HIGH)。
* **關閉 (Clear Bit 0)**：`REG &= ~(1 << n);`
  * 先把 1 左移，然後 `~` 反轉變成 `11110111`，再用 `&` (AND) 強制把目標 bit 壓成 0。

## 2. 巨集陷阱再進化：括號的防護網
如果要寫出一個反轉位元的巨集，絕對不能漏掉內部參數的括號：
```c
// 正確寫法
#define INVERT_BIT(X, n) ((X) ^= (1 << (n)))
```
* **原因**：C 語言中左移 `<<` 的優先權比 AND `&` 還高。如果不把 `n` 包起來，當呼叫 `INVERT_BIT(REG, a & b)` 時，會先算 `1 << a` 再跟 `b` 做 AND，引發毀滅性 Bug。

## 3. 系統廠神級白板題：判斷 2 的冪次方
* **核心觀念**：2 的冪次方在二進位中**永遠只有一個 1**。
* **O(1) 極速演算法**：`(x & (x - 1)) == 0`
* **原理解析**：當減去 1 時，唯一的 1 會發生借位變成 0，其後的 0 變成 1。拿原數字跟減一後的數字做 `AND`，就一定會完美錯開變成 `0`。

## 4. 實戰：特定範圍的數值修改 (Clear & Set)
在修改 32 位元暫存器的特定幾個 bits (例如設定第 12~15 個 bits) 時，必須分為兩步：
```c
// 步驟一：先清空該範圍為 0 (Clear)
// 0xF (二進位 1111) 左移 12 格，反轉後做 AND
REG &= ~(0xF << 12); 

// 步驟二：填入新數值 (Set)
// 把新數值 0x5 左移 12 格，做 OR
REG |= (0x5 << 12);
```
使用 `<<` 左移運算，不僅可以避免手動計算 `~0xFFFF0FFF` 等長串遮罩的失誤，更能讓程式碼具備極高的可讀性。
