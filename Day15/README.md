# Day 15: 終極大會考與一線大廠高頻面試題破解

恭喜完成 14 天的魔鬼特訓！在今天的最終大會考中，我們破解了多個連資深工程師都容易踩雷的陷阱，並深入探討了系統廠與 IC 設計廠的必考白板題。

## 1. 🚨 C 語言最危險的陷阱：運算子優先權
在二元搜尋的防溢位計算中，很多人會直覺寫出：
```c
int mid = low + (high - low) >> 1; // ❌ 致命錯誤
```
**原因**：C 語言中 `+` 的優先權高於 `>>`，這會導致程式先算出 `high`，再把 `high` 右移一半。
**破解法**：永遠用防護罩 (括號) 包好包滿！
```c
int mid = low + ((high - low) >> 1); // ✅
```

## 2. 🔄 32 位元整數反轉 (Reverse Bits)
* **直覺解法**：從外向內用 `(num >> i) & 1` 配合 `|=` 和 `&=` 互換。邏輯無懈可擊，但 CPU 分支預測效能較差。
* **👑 業界標準解法**：準備一個乾淨的 `result` 箱子，利用極速位移。
```c
uint32_t result = 0;
for (int i = 0; i < 32; i++) {
    result <<= 1;          // 🌟 關鍵：先左推騰出空間 (避免迴圈結束時多推一格)
    result |= (num & 1);   // 將最右邊的 bit 塞入
    num >>= 1;             // 捨棄剛才處理完的 bit
}
return result;
```
空間複雜度是完美的 **$O(1)$**。

## 3. 🔀 字串原地反轉 (In-Place String Reversal)
* 面試官最愛看的「純指標操作 (Double Pointers)」。
* 宣告 `start` 指標站頭，`end` 指標站尾。
* 終止條件：`while (start < end)`。這是在比較「實體記憶體位址的大小」。一旦位址交錯或相撞，迴圈立刻停止，完美相容奇數與偶數長度！

## 4. 🧱 記憶體對齊與 Padding (Memory Alignment)
* 結構體的大小並非單純變數大小相加。為了讓 CPU 高效抓取資料，編譯器會在變數之間塞入 Padding (補零)。
* **對齊法則**：每個變數的起始地址必須是自己大小的倍數。整個結構體的對齊值，取決於內部「佔用空間最大」的變數。
* **極限優化 (Struct Packing)**：把佔用空間最大的變數 (如 `int`) 往前擺，能把 Padding 擠出結構體，達成零浪費！

## 5. 🚂 單向串列的「安全刪除」節點
* **標準解法**：使用 `prev` 和 `curr` 兩個指標一前一後巡邏。
* **🚨 致命邊界條件**：若刪除的是「第一節車廂 (Head)」，此時 `prev` 是 `NULL`。如果不特判 `if(prev == NULL)` 而直接執行 `prev->next`，會瞬間引發 **Segmentation Fault** 當機！
* **大神寫法 (Linus Torvalds)**：直接使用指標的指標 `Node_t **indirect` 對接位址，直接消滅 `Head` 的特殊邊界判斷，被譽為最高品味的 C 語言寫法！
