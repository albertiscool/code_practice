# Day 9: 面試終極白板題 - 反轉連結串列與遞迴思維

## 1. 反轉連結串列 (Reverse Linked List)
這是資訊界出題率永遠第一名的經典題目（LeetCode 206）。完美解法需要派出三個指標 (`prev`, `curr`, `next_node`) 同時合作，執行「四步舞曲」：
1. **記住下一站**：`next_node = curr->next;` (保命機制，防遺失)
2. **繩子往回綁**：`curr->next = prev;` (真正的反轉動作)
3. **前一站標示牌前進**：`prev = curr;`
4. **現在站標示牌前進**：`curr = next_node;`

## 2. 空間複雜度 O(1) 的迷思
在反轉串列的過程中，雖然我們宣告了 3 個臨時指標變數（消耗了 24 Bytes），但這 24 Bytes 是**固定的常數**。無論串列有 3 個節點還是 100 萬個節點，我們都不需要額外申請更多的記憶體。因此，空間複雜度是完美的 **O(1)**。

## 3. Pass-by-Value 的遺失慘案
如果反轉函式不使用雙重指標，那它必定要回傳「新的火車頭位址」，且 `main` 函式必須去接住它：
```c
my_list = reverse_list(my_list);
```
如果忘記接住回傳值，`my_list` 會依然傻傻指著原本的火車頭（反轉後已經變成火車尾，其 `next` 為 `NULL`）。這會導致印出串列時只剩一個節點，而其他所有的節點都化為幽靈車廂，造成嚴重的**記憶體外洩 (Memory Leak)**。

## 4. 遞迴 (Recursion) 的「無恥工作分配法」
遞迴是許多工程師的夢魘，要破解它，必須放棄在腦中推演所有的步驟，改用工作分配的思維：
1. **終止條件**：最小單位是什麼？（例如：剩下 1 個盤子 / 剩下 1 節車廂）。
2. **信任分身 (Delegate)**：把 N-1 的龐大工作丟給小弟（遞迴呼叫），並且**無條件相信他會完美做好**。
3. **做好自己的小工作**：專注把當前這個節點的收尾動作做好就好（例如：把最大的盤子移過去 / 把下一節車廂往回牽）。
* 經典案例：遞迴版反轉串列、河內塔 (Tower of Hanoi)。
